# [Feature Spec] 02. 프로젝트 관리 ERP (Project Management ERP)

## 1. 도메인 개요 (Domain Overview)
본 모듈은 개발 프로젝트의 전 생애주기(Life-cycle)를 관리하는 통합 생산 관리 시스템입니다. 단순한 작업 추적을 넘어, 비즈니스 측면의 **계약-수행-배포-수익 정산-평가**로 이어지는 워크플로우를 디지털화하여 개발사의 운영 효율성을 극대화하는 것을 목적으로 합니다.

- **핵심 가치**: 투명한 진척도 관리, 정교한 수익 추적, 데이터 기반의 개발자 역량 평가.
- **대상 사용자**: PM(프로젝트 매니저), 개발자, 외부 파트너사, 최종 고객.

---

## 2. 화면 정의 (Page Definitions)

### 2.1 ERP 종합 대시보드 (Overall Dashboard)
진행 중인 모든 프로젝트의 현황을 한눈에 파악하는 통합 관제 화면입니다.
- **상태 요약**: 활성 프로젝트 수, 전체 진척률, 이번 달 예상 수익, 긴급 이슈 건수.
- **프로젝트 리스트 카드**: 각 프로젝트별 상태(정상/지연/위험), 마일스톤 달성도, 잔여 기간 표시.
- **수익 현황 위젯**: 월별 매출 및 비용 추이 그래프.
- **알림 섹션**: 컨펌 대기 중인 계약, 오늘 마감인 마일스톤, 최근 배포 이력.

### 2.2 프로젝트 상세: 일지 및 마일스톤 (Project Detail)
개별 프로젝트의 실행 단계를 관리하는 핵심 화면입니다.
- **마일스톤 탭**: WBS(Work Breakdown Structure) 기반의 일정 관리. Gantt 차트와 연동.
- **개발 일지(Journal) 탭**: 일일 업무 내용 기록. 태스크별 소요 시간 및 관련 이슈 링크.
- **진척도 뷰**: 마일스톤 대비 실제 일지 기록을 바탕으로 자동 계산된 달성률 표시.

### 2.3 계약 및 외주 관리 (Contract & Outsourcing)
비즈니스 법무 및 파트너 협업을 관리합니다.
- **계약서 관리**: 표준 계약서 템플릿 선택 및 작성. 전자 서명 상태 추적.
- **외주 현황**: 투입 인력 명단, 계약 금액, 기성고(중도금) 지급 일정 관리.
- **컨펌 워크플로우**: 산출물 업로드 및 고객/내부 승인 프로세스 관리(승인/반려/재검토).

### 2.4 수익 및 평가 (Revenue & Evaluation)
종료된 프로젝트의 성과를 분석하고 인사이트를 도출합니다.
- **프로젝트 수익 정산**: 투입 리소스(인건비) 대비 매출 분석. ROI(투자 대비 효율) 리포트.
- **개발자 역량 평가**: 일정 준수율, 코드 품질 지표(정적 분석 연동), 협업 만족도 등을 점수화.
- **마케팅/로그 분석**: 서비스 런칭 후 유입 경로 및 접속자 통계 데이터 요약.

## 3. 핵심 UI 구성 요소 (Key Components)

| 구성 요소 | 기능 설명 | 비고 |
| :--- | :--- | :--- |
| **인터랙티브 간트 차트** | 마일스톤 간 의존성 표시, 드래그 앤 드롭 일정 조정. | D3.js 또는 전용 라이브러리 |
| **마크다운 일지 에디터** | 코드 스니펫 지원, 이미지 업로드, 템플릿 선택 기능. | Toast UI Editor 등 |
| **진척도 진행바 (Progress)** | 목표치 대비 실적을 시각화. 지연 시 색상 변경(Red/Yellow). | |
| **수익/지출 추이 그래프** | 매출, 비용, 순이익을 다중 레이어로 표시. | Chart.js 또는 Recharts |
| **컨펌 상태 배지** | 미확인, 승인 대기, 반려, 승인 완료 상태 표시. | 디자인 시스템 준수 |

## 4. 사용자 인터랙션 (Interactions)

### 4.1 일지-진척도 자동 연동 (Auto-Sync)
- 개발자가 개발 일지에서 특정 마일스톤의 태스크를 '완료'로 표시하면, 해당 프로젝트의 마일스톤 진척도가 실시간으로 업데이트됩니다.

### 4.2 컨펌 워크플로우 (Approval Workflow)
1. **제출**: 개발자/PM이 산출물(디자인/코드/문서)을 등록하고 컨펌 요청.
2. **검토**: 승인권자(고객/상급자)에게 알림 발송.
3. **결정**: 승인(다음 단계 진행) 또는 반려(피드백 첨부 및 재수정 요청).

### 4.3 멀티 프로젝트 스위칭 (Multi-Project Switching)
- 좌측 상단 또는 사이드바의 프로젝트 셀렉터를 통해 컨텍스트를 유지하면서 다른 프로젝트의 ERP 화면으로 즉시 전환 가능.

## 5. 데이터 요구사항 (Data Requirements)

### 5.1 Project Schema
- `id`, `name`, `client_info`, `budget`, `start_date`, `end_date`, `status` (Planning/In-Progress/Paused/Completed).

### 5.2 Milestone & Task
- `project_id`, `title`, `description`, `weight` (가중치), `due_date`, `is_completed`.

### 5.3 Revenue Statistics
- `monthly_revenue`, `monthly_expense`, `profit_margin`, `developer_cost_breakdown`.

### 5.4 Evaluation Metrics
- `developer_id`, `reliability_score` (일정), `quality_score` (코드), `peer_review_text`.
