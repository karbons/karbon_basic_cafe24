## 2. 개요 (Overview)

### 2.1 목적
카본빌더의 AI 및 온톨로지 모듈은 개발자에게 차세대 지능형 개발 보조 환경을 제공하는 것을 목적으로 합니다. 특히 고가의 클라우드 GPU 비용 부담을 줄이기 위해 로컬 리소스를 공유하고, 프로젝트의 고유한 컨텍스트와 데이터 관계(온톨로지)를 AI가 깊이 이해할 수 있도록 지원합니다.

### 2.2 비전: 공유 AI GPU 품앗이
"나의 유휴 자원을 나누고, 필요할 때 집단 지능을 활용한다."  
개발자 PC의 유휴 GPU 자원을 상호 공유하여, 대규모 로컬 LLM(Language Model)이나 멀티 에이전트를 고비용의 외부 API 없이도 원활하게 구동할 수 있는 탈중앙화된 AI 생태계를 구축합니다.

---

## 3. 핵심 기능 (Core Features)

### 3.1 카본 공유 AI (GPU 품앗이)
- **자원 풀링**: 사용자의 PC가 유휴 상태일 때 GPU 자원을 카본 네트워크에 제공.
- **분산 연산**: 멀티 에이전트 구동 시 타 회원의 PC 자원을 활용하여 로컬 LLM 추론 속도 향상.
- **P2P 연결**: 보안이 강화된 P2P 방식으로 GPU 노드 간 연결 및 데이터 전송.

### 3.2 AI 컨텍스트 관리 (Context Management)
- **코드 컨텍스트**: 현재 프로젝트의 파일 구조, 함수 간의 의존성, 변수 타입 정보를 AI에게 주입하여 코드 생성 및 리뷰의 정확도 극대화.
- **디자인 컨텍스트**: 스토리보드, 디자인 시스템, 컴포넌트 정의 정보를 AI와 공유하여 UI/UX 구현 지원.
- **실시간 인덱싱**: 코드나 디자인 변경 시 실시간으로 컨텍스트 벡터 데이터베이스를 업데이트.

### 3.3 온톨로지 관리 (Ontology Management)
- **데이터 관계 정의**: 프로젝트 내 데이터 간의 의미론적 관계(Semantics)를 정의.
- **도메인 지식 체계화**: 단순 DB 스키마를 넘어, 비즈니스 로직 상의 객체 간 관계를 온톨로지로 구성하여 AI의 추론 능력 향상.
- **시각화**: 정의된 온톨로지를 그래프 형태로 시각화하여 데이터 흐름과 관계를 직관적으로 파악.

## 4. 사용자 시나리오 (User Scenarios)

### 시나리오 1: GPU 자원 기여 및 활용
1. 개발자가 카본빌더 설정에서 'GPU 품앗이 참여'를 활성화합니다.
2. 개발자가 자리를 비우거나 작업량이 적을 때, 시스템은 유휴 GPU 자원을 네트워크에 할당합니다.
3. 이후 개발자가 복잡한 멀티 에이전트 태스크를 수행할 때, 네트워크의 다른 유휴 자원을 빌려와 로컬 LLM을 고속으로 실행합니다.

### 시나리오 2: 컨텍스트 기반 코드 생성
1. 개발자가 새로운 API 기능을 구현하려고 합니다.
2. AI는 'AI 컨텍스트 관리' 모듈을 통해 기존의 DB 스키마(Supabase 방식)와 기존에 작성된 유사한 라우트 파일을 읽어옵니다.
3. AI는 프로젝트의 코딩 스타일과 온톨로지 정의에 맞게 최적화된 코드를 제안합니다.

## 5. 기술적 요구사항 (Technical Requirements)

### 5.1 백엔드 및 연산
- **Rust Axum**: 고성능 연산 및 동시성 처리를 위해 Rust 기반의 확장 백엔드 구축.
- **Local LLM**: Llama 3, Mistral 등 오픈소스 모델과의 연동 인터페이스 제공.
- **Vector DB**: 컨텍스트 검색을 위한 경량 벡터 데이터베이스(예: LanceDB, Qdrant 등) 활용.

### 5.2 통신 및 보안
- **P2P 통신**: GPU 자원 공유를 위한 WebRTC 또는 유사 기술 기반의 보안 연결.
- **데이터 암호화**: 공유되는 연산 데이터의 보안을 위해 엔드투엔드(E2EE) 암호화 적용.

### 5.3 프론트엔드
- **SvelteKit 기반 관리 UI**: 공유 현황 모니터링, 컨텍스트 인덱싱 상태 확인, 온톨로지 편집 도구 제공.
