# Feature Spec 05: AI 및 온톨로지 (AI & Ontology)

## 1. 도메인 개요 (Domain Overview)
본 모듈은 개발자에게 차세대 지능형 개발 보조 환경을 제공하며, 특히 고비용의 클라우드 GPU 자원 의존도를 낮추기 위한 'GPU 공유 비전(품앗이)'을 핵심으로 합니다. 사용자의 유휴 GPU 자원을 네트워크에 공유하고 필요시 타 회원의 자원을 빌려 쓰는 탈중앙화된 AI 생태계를 구축합니다. 또한, 프로젝트의 코드, 디자인, 데이터 간의 의미론적 관계(온톨로지)를 AI가 깊이 이해할 수 있도록 컨텍스트를 체계적으로 관리하여 코드 생성 및 추론의 정확도를 극대화합니다.

---

## 2. 화면 정의 (Page Definitions)

### 2.1 AI 공유 대시보드 (AI Sharing Dashboard)
- **목적**: GPU 자원 공유 현황 모니터링 및 네트워크 연결 관리.
- **주요 기능**:
    - **자원 현황**: 로컬 GPU 사용률, 온도, 메모리 상태 및 네트워크 전체 가용 자원 표시.
    - **공유 설정**: GPU 품앗이 참여 토글(On/Off) 및 할당할 자원 임계치 설정.
    - **네트워크 상태**: P2P 연결 노드 수, 지연 시간(Latency), 데이터 전송 속도 확인.
    - **활동 로그**: 자원 기여 시간 및 활용 내역, 획득/사용 포인트(크레딧) 요약.

### 2.2 컨텍스트 관리자 (Context Manager)
- **목적**: AI가 학습/참조할 프로젝트 데이터 범위를 지정하고 인덱싱 상태 관리.
- **주요 기능**:
    - **소스 선택**: 코드 파일, 폴더, DB 스키마, 디자인 문서 등 컨텍스트 포함 여부 설정.
    - **벡터 DB 상태**: 실시간 임베딩 인덱싱 진행률 및 벡터 데이터베이스 동기화 상태 표시.
    - **인덱스 제어**: 수동 인덱싱 트리거, 특정 데이터 제외(Ignore) 필터링.
    - **검색 테스트**: 등록된 컨텍스트를 기반으로 한 시맨틱 검색 결과 미리보기.

### 2.3 온톨로지 에디터 (Ontology Editor)
- **목적**: 데이터 간의 의미론적 관계를 시각적으로 정의하고 관리.
- **주요 기능**:
    - **그래프 뷰**: 엔티티(Entity)와 관계(Relation)를 노드와 간선으로 시각화.
    - **스키마 매핑**: DB 테이블을 온톨로지 노드로 변환하고 비즈니스 로직 상의 관계 설정.
    - **속성 정의**: 각 엔티티의 의미론적 설명(Metadata) 및 추론을 위한 규칙(Rules) 추가.
    - **내보내기**: 정의된 온톨로지를 JSON, RDF 또는 AI 프롬프트용 지식 베이스로 추출.

## 3. 핵심 UI 구성 요소 (Key Components)

- **GPU 리소스 모니터 (GPU Resource Monitor)**: 실시간 GPU 상태(로드, 온도)를 차트와 게이지 형태로 보여주는 대시보드 위젯.
- **벡터 인덱싱 상태 바 (Vector Indexing Status Bar)**: 백그라운드에서 진행되는 컨텍스트 인덱싱 작업의 진행률과 상태를 표시하는 프로그레스 바.
- **온톨로지 관계 그래프 (Ontology Relation Graph)**: 엔티티 간의 복잡한 관계를 드래그 가능한 노드와 연결선으로 표현하는 인터랙티브 캔버스.
- **컨텍스트 셀렉터 (Context Selector)**: 프로젝트 트리 구조에서 AI가 인지해야 할 대상을 체크박스로 선택하는 탐색기 UI.
- **P2P 노드 맵 (P2P Node Map)**: 현재 연결된 공유 AI 네트워크의 노드 분포를 시각화한 미니 맵.

---

## 4. 사용자 인터랙션 (Interactions)

- **GPU 공유 토글**: 스위치 클릭 시 즉시 P2P 네트워크에 노드로 등록되거나 자원 공유를 중단합니다.
- **실시간 인덱싱 제어**: 파일 변경 시 자동으로 인덱싱이 수행되거나, 사용자가 '지금 인덱싱' 버튼을 눌러 강제로 업데이트할 수 있습니다.
- **시각적 관계 정의 (Node Connection)**: 온톨로지 에디터에서 노드와 노드를 선으로 연결하고 팝업을 통해 관계의 성격(예: '속함', '의존함', '확장함')을 정의합니다.
- **자원 할당 슬라이더**: AI 연산에 사용할 최대 GPU 사용량(%)이나 메모리 제한을 슬라이더로 조절합니다.

---

## 5. 데이터 요구사항 (Data Requirements)

- **GPU Stats**:
    - 로컬/피어 GPU 모델명, 사용률(%), 온도, VRAM 사용량.
- **Context Metadata**:
    - 파일 경로, 해시값(변경 감지용), 인덱싱 완료 시간, 벡터 임베딩 모델 정보.
- **Ontology Graph JSON**:
    - Nodes: ID, Label, Properties (DB Table, Description).
    - Edges: Source ID, Target ID, Relationship Type (Is-A, Has-A, etc.).
- **Network Info**:
    - P2P Peer List, 노드별 가용성, 연결 지연 시간(ms), 암호화 키 정보.
- **Usage Metrics**:
    - 기여 자원량(TFLOPS-Hours), 네트워크 활용도, 누적 크레딧 정보.
