# PRD 03: 데이터베이스 및 모델링 (DB & Modeling)

## 1. 개요 (Overview)
본 모듈은 카본빌더의 '데이터 중심 개발' 원칙을 실현하기 위한 핵심 도구입니다. 개발자가 복잡한 SQL 명령어나 외부 도구 없이도 웹 GUI 환경에서 직관적으로 데이터베이스를 설계하고, 이를 시각화된 ERD로 관리하며, 프론트엔드 및 백엔드 개발에 즉시 사용 가능한 타입 정의를 자동 생성하는 것을 목적으로 합니다.

### 비전
- "DB 설계가 곧 코드 개발의 시작"이 되는 심리스(Seamless)한 개발 환경 제공.
- Supabase의 편리함과 로컬 개발의 유연함을 결합한 DB 관리 경험 제공.

---

## 2. 핵심 기능 (Core Features)

### 2.1 Supabase 방식 DB 관리
- **테이블 빌더**: SQL 작성 없이 UI를 통해 테이블 생성, 컬럼 추가, 데이터 타입 지정 및 제약 조건(PK, FK, Not Null 등) 설정.
- **데이터 브라우저**: 엑셀 형태의 그리드 UI를 통해 테이블 내 데이터를 직접 조회, 추가, 수정, 삭제.
- **스키마 동기화**: UI에서 변경된 사항을 실제 PostgreSQL 또는 MySQL 데이터베이스에 즉시 반영.

### 2.2 ERD 다이어그램 관리
- **자동 시각화**: 현재 DB 구조를 분석하여 테이블 간 관계를 ERD 다이어그램으로 자동 생성.
- **드래그 앤 드롭**: 다이어그램 상에서 테이블 위치 조정 및 관계(Relationship) 시각화.
- **내보내기 기능**: 설계된 ERD를 이미지 또는 문서 형태로 내보내어 협의 시 활용.

### 2.3 변수 모델 및 타입 정의
- **TypeScript 타입 자동 생성**: DB 스키마를 기반으로 프론트엔드(SvelteKit) 및 백엔드에서 사용할 인터페이스(Interface) 및 타입(Type) 자동 생성.
- **DTO(Data Transfer Object) 정의**: API 통신에 최적화된 데이터 모델 정의 및 검증 로직 연결.
- **온톨로지 연동**: 데이터 간의 의미적 관계를 정의하고 이를 AI 컨텍스트로 활용할 수 있도록 지원.

---

## 3. 사용자 시나리오 (User Scenarios)

1. **DB 설계**: 개발자는 카본빌더 관리자 UI의 'DB 관리' 메뉴에 접속하여 새로운 프로젝트용 테이블을 생성합니다.
2. **관계 설정**: 'ERD 다이어그램' 뷰로 전환하여 테이블 간의 외래키 관계를 시각적으로 확인하고 조정합니다.
3. **데이터 입력**: 테스트를 위해 데이터 브라우저에서 몇 가지 샘플 데이터를 직접 입력합니다.
4. **타입 적용**: '타입 정의' 버튼을 클릭하여 생성된 TypeScript 파일을 프론트엔드 프로젝트의 `src/lib/types` 폴더로 저장하고 코드 내에서 즉시 사용합니다.

---

## 4. 기술적 요구사항 (Technical Requirements)

### 4.1 인프라 및 호환성
- **지원 DB**: PostgreSQL, MySQL (그누보드5 기본 DB 지원 포함).
- **연동 방식**: Docker로 구성된 DB 컨테이너와 직접 연결하여 메타데이터 추출.

### 4.2 인터페이스
- **Frontend**: SvelteKit 기반의 반응형 웹 UI.
- **상태 관리**: 실시간 스키마 변경 사항 반영을 위한 웹소켓 또는 최적화된 API 통신.

### 4.3 보안
- **권한 제어**: 관리자 계정만 DB 구조 변경이 가능하도록 제한.
- **백업 및 복구**: 스키마 변경 전 스냅샷 생성 및 롤백 기능 지원.

---

## 5. 수용 기준 (Acceptance Criteria)

- [ ] 웹 UI를 통해 신규 테이블을 생성하고 컬럼을 추가할 수 있는가?
- [ ] 생성된 테이블이 실제 DB(PostgreSQL/MySQL)에 올바르게 반영되는가?
- [ ] ERD 다이어그램에서 테이블 간의 외래키 관계가 올바르게 표시되는가?
- [ ] DB 스키마 변경 시 TypeScript 타입 파일이 오류 없이 자동 생성되는가?
- [ ] 데이터 브라우저에서 데이터의 CRUD 작업이 정상적으로 작동하는가?
- [ ] 여러 프로젝트의 DB 환경을 개별적으로 관리할 수 있는가?
