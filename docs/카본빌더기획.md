# Karbon Builder 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트 배경
Karbon Builder는 산업 현장에서 AI를 쉽게 도입할 수 있는 ERP(Enterprise Resource Planning) 보일러플레이트를 제공하는 오픈소스 프로젝트입니다. 기존의 복잡한 개발 도구와 달리, GnuBoard처럼 초보자도 접근하기 쉽게 설계되었으며, 하이브리드 앱과 AI 기능을 통합합니다. 이 프로젝트는 개발의 용이성, 비용 효율성, 그리고 공유 경제 모델을 핵심으로 하여, 중소기업의 AI 이식율을 높이는 것을 목표로 합니다.

### 1.2 프로젝트 범위
- MVP(Minimum Viable Product) 단계: PHP 기반으로 빠른 프로토타이핑.
- 엔터프라이즈 단계: Rust MSA(Microservices Architecture)로 전환하여 스케일업.
- 주요 기능: ERP 모듈(회원 관리, RBAC, PII 보호, S.T.O), AI 온톨로지 개발 환경, AI 공유 API(개인 PC GPU 임대 모델).
- 타겟 사용자: 산업 현장 관리자, 중소기업 개발자, AI 도입을 원하는 제조/물류 업체.

### 1.3 기대 효과
산업 전반의 AI 도입률 향상(현재 20-30%에서 70% 이상 목표), 개발 비용 절감, 개인 소득 창출(공유 경제 모델 통해).

## 2. 목표와 비전

### 2.1 단기 목표
- MVP 개발: PHP 기반으로 1-2개월 내 PoC(Proof of Concept) 완성.
- 기능 통합: 파일 기반 API 라우팅, JWT 인증, 기본 ERP 모듈 구현.
- 테스트: 산업 현장 파일럿 테스트(중소 제조업체 5곳 대상).

### 2.2 장기 비전
- 엔터프라이즈 전환: Rust MSA로 고성능/안정성 확보.
- AI 공유 경제 확대: 개인 PC GPU를 Airbnb 스타일로 임대, AI 추론 비용 1/10로 절감.
- 커뮤니티 성장: GitHub 오픈소스 공개, 기여자 유치.
- 사회적 영향: AI로 인한 일자리 상실 방지 – 개인 PC를 활용한 소득 개인화 모델.

### 2.3 균형적 접근
개발 편의성(쉬운 시작)과 성능(스케일업)을 균형 있게 고려. AI 기능은 공유 경제로 비용 부담을 최소화하되, 보안(JWT 등)을 최우선으로. 사회에서 ai로 인한 소득 불평등 해소로 ai 확산 대한 저항을 해소

## 3. 기술 스택과 아키텍처

### 3.1 MVP 기술 스택 (PHP 중심)
- 백엔드: PHP 8+ (함수형 + 싱글턴 혼합, PDO for DB).
- 프론트엔드: Svelte + Carbon UI Builder (드래그-앤-드롭 UI 조합).
- 앱: Capacitor (하이브리드 iOS/Android 지원).
- DB: PostgreSQL (안전 쿼리 중심).
- API: 파일 기반 동적 라우팅 (SvelteKit 영감, 폴더/파일 생성으로 자동 매핑).

### 3.2 엔터프라이즈 기술 스택 (Rust 중심)
- 백엔드: Rust + Axum (파일 기반 라우팅 크레이트 활용, e.g., axum-folder-router).
- DB: SQLx (타입-세이프 쿼리, PHP 쿼리와 호환).
- MSA: Docker Compose/Kubernetes로 모듈 분리 (user-service, ai-service 등).
- 공통: OpenAPI yaml로 API 스펙 공유, config 스위치로 PHP ↔ Rust 전환.

### 3.3 아키텍처 설계
- 듀얼 스택: MVP는 PHP monolith, 전환 시 Rust MSA로 seamless 업그레이드.
- 라우팅: 파일 기반으로 직관적 (PHP: include & 함수 호출, Rust: macro 자동 등록).
- 인증: JWT 모듈 (수개월 개발 완료, access/refresh token, RBAC 통합).
- AI 통합: 온톨로지 에디터 + 공유 API (P2P 네트워크, blockchain 기반 결제 옵션).

폴더 구조 예:
```
karbon-builder/
├── php-mvp/      # PHP 코드
├── rust-msa/     # Rust 코드
├── routes/       # 공통 라우팅 파일 (api/users/[id].php or route.rs)
├── lib/          # 공유 헬퍼 (Database.php/db.rs)
└── docker-compose.yml  # 배포 지원
```

## 4. 주요 기능 상세

### 4.1 API 라우팅 시스템
- SvelteKit 스타일 파일 기반: 폴더 생성으로 엔드포인트 자동 생성 (e.g., api/users/[id].php → /api/users/123).
- PHP: 함수형 핸들러 (handle_get(), include 방식).
- Rust: axum-folder-router로 동일 패턴 유지.
- 장점: 개발 속도 향상, 초보자 친화적. 단, 동적 파라미터 매칭은 간단 regex로 처리.

### 4.2 JWT 인증 모듈
- 핵심 보안 기능: 토큰 생성/검증, claims 기반 RBAC.
- 구현: access token (short-lived), refresh token (revocation 지원, Redis 블랙리스트).
- 미들웨어: PHP/Rust 공통으로 요청 전 체크 (401/403 에러 핸들링).
- 개발 완료 상태: 수개월 투자로 프로덕션 레벨 안정성 확보, PII 보호 통합.

### 4.3 ERP 모듈
- 회원 관리: JWT 연동 로그인/등록.
- RBAC: 역할 기반 접근 (admin/user 등).
- PII 보호: GDPR 준수 암호화.
- S.T.O (Session+Task+Order): AI 기반 AICC (AI Contact Center) 통합, 세션/태스크/오더 관리.

### 4.4 AI 기능
- 온톨로지 개발 환경: Protégé 스타일 에디터, Rust로 API 래퍼.
- 공유 API: 개인 PC GPU 임대 (P2P, Vast.ai 영감). 임대자 수익 창출, 사용자 저비용 추론.
- 균형: AI 도입 용이성 강조, 하지만 보안(JWT)과 비용(공유 경제) 균형.

### 4.5 하이브리드 앱 통합
- Capacitor로 웹 → 네이티브 변환.
- 현장 적용: 모바일에서 ERP/AI 접근 (e.g., 센서 데이터 AI 분석).

## 5. 개발 로드맵과 리스크 관리

### 5.1 로드맵
- 1단계 (1-2개월): PHP MVP 개발 (라우팅 + JWT + 기본 ERP).
- 2단계 (2-3개월): AI 공유 API 프로토타입, 파일럿 테스트.
- 3단계 (3-6개월): Rust 전환, MSA 배포, 오픈소스 공개.
- 4단계 (6개월+): 커뮤니티 피드백 반영, 산업 협회 협업.

### 5.2 잠재적 리스크와 대안
- 리스크 1: 개발 복잡도 증가 (라우팅/인증 불균형) → 균형 설계로 사전 방지, 모듈화 우선.
- 리스크 2: 보안 취약점 (JWT 탈취) → 정기 감사 + 베스트 프랙티스 적용 (이미 완료된 모듈 활용).
- 리스크 3: AI 공유 네트워크 안정성 → P2P 라이브러리 테스트, fallback으로 클라우드 옵션.
- 리스크 4: 사용자 채택 저조 → GnuBoard 스타일 쉬운 인터페이스 강조, 무료 보일러플레이트 제공.
- 예산/자원: 초기 개발 비용 최소화 (오픈소스 도구), 정부 지원(AI 바우처) 활용.

## 6. 결론과 기대

Karbon Builder는 개발의 용이성과 AI의 접근성을 균형 있게 결합한 프로젝트로, 산업 현장의 혁신을 이끌 잠재력을 지녔습니다. 파일 기반 라우팅으로 빠른 시작, JWT로 강력한 보안, 공유 경제로 비용 효율성을 달성함으로써, 한국 산업의 AI 도입을 가속화할 것입니다. 이 기획은 모든 측면을 균형 있게 고려하여 지속 가능한 성장을 목표로 합니다. 다음 단계로 MVP 구현을 제안하며, 추가 논의 환영합니다.
