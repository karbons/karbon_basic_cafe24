# 카본빌더(Carbon Builder) 개요

## 1. 출발점 (Problem Statement)

그누보드5 는
한국 웹 생태계에서 가장 널리 사용되는 CMS 중 하나다.

그러나 스마트폰 시대가 10년 이상 지난 현재까지도:

* 앱(App)을 위한 공식 API 구조 부재
* 모바일/웹 분리 아키텍처 부재
* 현대적인 백엔드 확장 구조 부재

라는 한계를 가지고 있다.

다른 대안들:

* Laravel
* WordPress

는 API 기능을 제공하지만,

* 무거운 구조
* 높은 러닝커브
* 국내 저변 부족
* 과도한 프레임워크 의존성

때문에 “누구나 쉽게 만드는 CMS”라는 목적과는 거리가 있다.

즉,

> “앱을 쉽게 만들 수 있는 가벼운 CMS/백엔드 프레임워크는 존재하지 않는다”

이 문제의식에서 **카본빌더**는 시작되었다.

---

# 2. 카본빌더란 무엇인가

카본빌더는

> **파일 기반 API 프레임워크 + 경량 풀스택 아키텍처**

이다.

복잡한 엔터프라이즈 구조 대신,

* 단순함
* 가벼움
* 직관성
* 확장성

을 우선한다.

목표는 단 하나:

> “가장 적은 코드와 비용으로, 가장 멀리 확장 가능한 구조”

---

# 3. 핵심 철학 (Philosophy)

## 3.1 단순함이 곧 생산성이다

* 파일 기반 라우팅
* 함수 중심 설계
* 클래스 남발 금지
* 최소한의 규칙

누구나 10분 안에 이해할 수 있어야 한다.

코드는 설명이 아니라 **직관적으로 읽혀야 한다.**

---

## 3.2 프레임워크는 도구이지 종교가 아니다

특정 스택에 종속되지 않는다.

* SaaS 종속 ❌
* 특정 벤더 종속 ❌
* 클라우드 강제 ❌

오픈소스 기반으로 **어디서든 실행 가능**해야 한다.

---

## 3.3 비용은 경쟁력이다

복잡한 스택은 곧 비용이다.

* Java + Spring
* React 대규모 구조
* 무거운 ORM

은 개발비/운영비/인건비를 불필요하게 증가시킨다.

카본빌더는:

* 적은 메모리
* 적은 서버
* 적은 코드
* 적은 인력

으로 동일한 결과를 만든다.

> 효율 = 실력

이라는 관점을 따른다.

---

## 3.4 가짜 노동을 제거한다

기술 스택을 복잡하게 만들어

* 인건비를 부풀리고
* 구조를 과대설계하고
* 불필요한 추상화를 만드는

“가짜 생산성”을 거부한다.

카본빌더는:

> 최소한의 기술로 최대의 결과

를 추구한다.

---

## 3.5 미래 확장을 전제로 설계한다

현재 편의만을 위한 구조는 장기적으로 기술 부채가 된다.

따라서 처음부터:

* MSA 전환 가능
* Rust 전환 가능
* 모바일 앱 확장 가능
* 무제한 트래픽 대응 가능

구조를 갖는다.

---

# 4. 기술 선택 철학

카본빌더는 “최신 기술”이 아니라
"**효율 대비 성능이 가장 높은 기술**"을 선택한다.

---

## Backend

### PHP (FAPI)

* 가장 쉬운 진입장벽
* 빠른 개발
* 낮은 비용
* 그누보드 생태계 활용

### Rust

* 고성능
* 저메모리
* 안정성
* MSA/대규모 트래픽 대응

👉 PHP → Rust 점진 전환 전략

---

## Frontend

### SvelteKit

* React 대비 코드량 감소
* 개발 속도 빠름
* 번들 작음
* 개발 및 유지보수 쉬움

👉 개발 효율 최대화

---

## Mobile

### Capacitor

* 웹 코드 기반
* Android/iOS 동시 지원
* 네이티브 기능 확장 가능

👉 앱/웹 단일 코드베이스

---

# 5. 설계 원칙

카본빌더는 다음 원칙을 따른다.

### 1. File First

파일 구조 = 라우팅 구조

### 2. Function First

클래스보다 함수

### 3. Domain First

도메인 단위 분리 → MSA 준비

### 4. Lightweight First

무거운 프레임워크 금지

### 5. Replaceable Stack

언제든 Rust/다른 기술로 교체 가능

---

# 6. 지향점

카본빌더가 만드는 세계는:

* 소수 인원으로도 SaaS 구축 가능
* 스타트업/1인 개발자도 ERP 운영 가능
* 대기업 수준 트래픽 대응 가능
* 비용 대비 최고의 생산성

즉,

> “작게 시작해서, 끝없이 확장 가능한 시스템”

이다.

---

# 7. 한 문장 정의

### 카본빌더는

> “가볍게 시작해, 무한히 확장 가능한 베이스플레이트”

이다.
